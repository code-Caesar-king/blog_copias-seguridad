Técnicas de Migración de Datos

10 de abril de 2026 Lectura de 8 min SENA – FAVA

Mover datos entre sistemas es una de las tareas más críticas en el ciclo de vida del software. Exploraremos en detalle las metodologías, técnicas y estrategias que hacen posible una migración exitosa, sin pérdida de información y con máxima integridad.

01 — Fundamentos

¿Qué es la migración de datos?

La migración de datos es el proceso mediante el cual volúmenes de información son trasladados desde sistemas existentes hacia otros sistemas destino. Este proceso abarca la extracción, transformación, carga y limpieza de los datos.

Algunas situaciones comunes que la desencadenan: pasar de Oracle a SQL Server, cargar archivos Excel a una base de datos transaccional, o actualizar la versión de un sistema que cambia la estructura de sus tablas.

"La migración de datos debe estar acompañada de un plan que contemple planeación, ejecución y validación de los datos migrados."

Todo proceso de migración se realiza primero en un entorno de pruebas. Solo cuando sea exitoso y aprobado por el cliente, se procede al ambiente de producción, acompañado de un ticket de servicio y un documento de paso a producción con rollback definido.

02 — Metodología

Las tres etapas del proceso

La metodología de migración de datos se estructura en tres grandes procesos que se ejecutan de forma secuencial y se retroalimentan entre sí.

Planeación

Define objetivos, alcance, estrategias, riesgos y factibilidad técnica y económica. Incluye cronograma, equipo humano, listas de chequeo y plan de contingencia. También contempla el análisis del esquema de datos actual versus el nuevo.

Migración

Es la ejecución operativa. Se instalan herramientas, se aplican técnicas como el mapeador de campos, se gestionan restricciones y triggers, y se controla la codificación de caracteres para evitar pérdidas de información.

Validación

Se verifican los resultados usando contadores de registros, reportes estadísticos y revisión por parte de los usuarios del sistema. La suma de registros exitosos y fallidos debe coincidir con el total original.

03 — Técnicas

Cuatro técnicas principales

Existen diversas técnicas para ejecutar una migración. La elección depende del motor de base de datos, el volumen de datos y los requerimientos del proyecto.

01

Sincronización de bases de datos

Hace que dos bases de datos sean equivalentes al mismo tiempo, copiando datos y metadatos desde el origen al destino mediante asistentes del SMBD configurados con particiones y seguridad.

02

Importación / exportación de archivos

Uso de archivos planos, CSV o XML para mover datos entre plataformas. El formato CSV usa coma como delimitador y es ideal para hojas de cálculo; XML usa etiquetas como estándar de intercambio estructurado.

03

Sentencias DML

Uso de instrucciones SQL (SELECT, INSERT, UPDATE, DELETE) para trasladar datos entre bases de datos a través de los entornos administrativos de los SMBD, generando scripts reutilizables.

04

Procedimientos ETCL

Diseño y construcción de procedimientos técnicos de Extracción, Transformación, Carga y Limpieza. Sigue pasos definidos: selección de fuente, comprensión, filtrado, transformación, movimiento y documentación.

04 — ETCL en detalle

El proceso ETCL paso a paso

Los procedimientos ETCL son la esencia de la migración técnica. Se diseñan a partir del análisis de la migración y el mapeo de campos, contemplando todos los requerimientos de transformación.

En la fase de extracción, se identifican las fuentes de datos con participación activa de usuarios del negocio y el equipo de IT. Comprender la granularidad (nivel de tablas, campos y cruces) es fundamental para evitar errores.

La transformación incluye consultar, filtrar con cláusulas WHERE, y convertir los datos según las diferencias entre tipos de origen y destino. Las transformaciones deben poder deshacerse para garantizar un proceso seguro.

Finalmente, la carga mueve los datos usando Transact-SQL, procedimientos almacenados u otras herramientas, y concluye con pruebas, auditoría y documentación específica según el cronograma del proyecto.

05 — Estrategias

¿Cuál estrategia elegir?

Cada proyecto de migración tiene condiciones únicas. Aquí las cuatro estrategias más usadas según el contexto del negocio.

Big-Bang

Se migra la totalidad de los datos en una sola operación durante una ventana de mantenimiento definida. El sistema anterior se apaga el día estipulado y todos los usuarios pasan al nuevo sistema desde esa fecha.

Ejecución en paralelo

Los dos sistemas corren simultáneamente durante un período de tiempo. Los usuarios trabajan en el antiguo y se familiarizan con el nuevo, garantizando continuidad antes de desactivar el sistema previo.

Migración incremental o por fases

El nuevo sistema se activa gradualmente conforme se completan las migraciones por módulos o fases. No se espera a tener todo listo para comenzar la operación parcial del sistema destino.

Cero tiempos de inactividad

Para procesos críticos que no pueden detenerse. Combina estrategias incrementales y paralelas para minimizar el tiempo de inactividad al mínimo posible, incluso en proyectos de meses de duración.
